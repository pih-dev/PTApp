Client Evaluation & Training System — Structure

Five subsystems. Build three now (blue), defer two (grey). Arrows show data flow.

Build now
1 — FOUNDATION

Exercise Library

Import from Telegram Excel. Each exercise tagged with type, muscle group, difficulty, equipment. Shared data used by eval and programs.

Build now
2 — INPUT

Evaluation System

Battery of timed tests (pushups, squats, situps, jumping jacks, step-ups…). Each scored 1–5: Weak → Excellent. Stored on the client record.

Build now
3 — LOGIC

Classification

Age + gender + eval scores → level. Levels: Beginner A/B, Intermediate A/B, Pro, Elite. Placeholder standards to start — tune later.

Later
4 — OUTPUT

Program Generator

Takes classification → daily/weekly/monthly plan for a year. ~3× per week. Mixes exercises appropriately for level. Deferred.

Later
5 — UX

Program UI / Workflow

How PT and clients see and use the generated program day-to-day. Deferred until generator exists to show something real.

DATA FLOW

1 + 2 → 3 → 4 → 5

Exercise library is foundation. Eval produces scores. Classification reads scores + age/gender → level. Later: generator reads level + library → plan. UI consumes plan.
